Description Andrew Clark 2014-07-07 04:51:24 CDT 
Just updated and found a strange bug. If run the following as an IJ1 Macro from the editor:

foo = "bar";
print(foo);

there is (of course) no problem. However, if I run

eoo = "bar";
print(eoo)

I get the following error: " Statement cannot begin with <=> in line 2. / <=> "bar"; "

I get this error for any variable beginning with the letter "e". It runs fine with variables starting with any other letter. If I put something valid in front of this statement (so that eoo="bar" is not on line 1), it runs fine. For example:

print("foo")
eoo = "bar";
print(eoo);

runs with no problem.

Hope it's any easy fix. Strange bug. Kind of funny though.

cheers,
Andy

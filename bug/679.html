Description John L 2013-11-25 14:12:03 CST 
It appears that there is a bug in how Fiji is handling the getDirectory command.  

Starting on 11/22 macros using getDirectory would no longer allow user selection from the OS of the directory where image sets were placed.  The user prompt appears for the first call for a path but no user prompt appears for the second call.  Occasionally the second user prompt will appear but as soon as the trackpad is touched the script continues and a null error is thrown since no directory is chosen.

 Installation of the latest daily Fiji build (1.48h10) did not solve the problem nor did attempting to run a simple test macro in ImageJ alone.

Test macro for degugging:
//getDirectory test 
dir1 = getDirectory("Choose Directory 1");
dir2 = getDirectory("Choose Directory 2");
print(dir1);
print(dir2);

It seems that the problem is with Java but that is just a guess.  Any help would be most appreciated while I try to gin up a work around.

Thanks so much.

John

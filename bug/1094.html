Comment 2 Curtis Rueden 2015-06-19 09:32:45 CDT 
Thanks for the report, and especially for the detailed investigation, Radoslaw.

I changed the Info.plist file on the ImageJ and Fiji update sites to reference ImageJ-macosx directly instead of the old ImageJ-tiger delegator. The ImageJ-tiger launcher was intended to facilitate continued support for old versions of OS X, but it has not been tested for years and since it is causing trouble on some systems, I'd rather just remove it from the equation.

I did leave ImageJ-tiger on the update site, so that anyone who might possibly still benefit from it can choose to do so (by invoking it directly, or by editing their Info.plist). But I think this scenario is unlikely.

Note that on my OS X Yosemite system, I do not have this problem. Maybe because I upgraded from previous OS versions? Or maybe because I also have Apple Java 6 installed. I am bad at reading dtruss output, so I made a gist in case you are curious:

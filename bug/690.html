Description bugz 2014-01-07 09:07:58 CST 
Color -> Merge Channels allows up to 7 images to be combined into a multi-channel stack.

This is great, except that it attempts to auto-assign images to each channel, which is super annoying when I have a dozen windows open and just want to merge two of them. Now I have to 5 selection to turn channels back to None, instead of 2 to select just the images I want.

This should not be the default behavior, if there is actually a scenario where this autofill is useful, how about a button on the dialog to carry it out?
Comment 1 Wayne Rasband 2014-01-08 09:09:42 CST 
In the latest ImageJ daily build (1.48p7), the Merge Channels command no longer attempts to auto-fill its dialog unless there are images created by the Split Channels command that have names starting with "C1-", "C2-", etc., or names ending with "(red)", "(green), etc.
Comment 2 Curtis Rueden 2014-01-08 13:35:20 CST 
Nice solution, Wayne. Closing the bug!

Description Aaron Ponti 2011-01-26 09:01:22 CST 
We used to have a simple macro in ImageJ that would open a text file (containing just one line with a series of parameters), get the content from the window and parse it. Something like:

open(fileName);
info=getInfo("window.contents");
// parse info ...

The problem in Fiji is that open() command now loads the file in the new script editor (instead of in one new window as in ImageJ), and therefore the

info=getInfo("window.contents");

line does not find it...

We solved the problem by using File.openAsString() (which is better), but still the behavior of open() in Fiji remains a bit confusing.

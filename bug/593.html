Description Eric Perlman 2013-04-23 09:23:52 CDT 
We often generate TrakEM2 projects where each of the patches are retrieved via http rather than the local file system.  At some point in the past few months, this functionality is now broken in a very weird way:  the image contents end replicated accross multiple patches.

Basically, if a project conains 10 patches, the first five will be a replica of the first patch, and the next five will be a replica of the second.

To reproduce:
- Patches must have a remote file path (file_path="http://...")
- n_mipmap_threads >= 2
- Patches must be "slow" to load (~ >= 1M)

I will attach a sample project file when I figure out how to do that. ;-)

I suspect the problem lies with ij.io.Opener.openURL() being called from multiple mipmap threads, but I haven't ruled out the TrakEM2 mipmap generation code either.

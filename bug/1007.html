Description unni 2015-02-18 10:09:02 CST 
I have 3325MB of RAM and a core 2 duo 64 bit windows system running windows 7 ultimate OS.
When I use the plugin "plugin register virtual stack slices", I observe that the registered slices are
saved one by one in my selected output directory but the saving order is not numerically sequential.
It saved files NEF1, NEF2,NEF3,NEF4 and suddenly becomes NEF10,NEF11,etc. The screen shot
of the directory , I will try to upload. It shows the sequence of saving. There were 60 files in the input directory
each of 23MB Tiff format. When the 59th file was saved, I saw a log window as below:
>>>>>>>>>>>>>>>>>>>>>>>>>>>
<Out of memory>
<All available memory (2995MB) has been>
<used. To make more available, use the>
<Edit>Options>Memory & Threads command.>
>>>>>>>>>>>>>>>>>>>>>>>>>>>
My queries :

1. Being virtual stack, I hoped that there is no memory limit and I could process 100 or 200 such files. Am I right ?
2. Why the files are not saved in the numerical order ? If so, file NEF5 would not have got distorted.
3. I feel that as the processing of about 40 files were over, memory trouble began and subsequent files were 
    wrongly processed before giving out the memory shortage error in log window.
4. I tried the virtual stack in some of the other menus but the effect is the same.
5. My requirement is to process 100 or 200 such files for use in focus stacking. Since I did not find a method to  
    combine the stack , I use the aligned images from FIJI as input in a program COMBINE ZP and the results are 
    excellent. 
6. So I processed only 45 files using virtual stack method instead of my available 70 files. There was no error.
    Using the normal method of stacking into RAM, I can process only 24 files maximum, in my system. (Linear stack 
   alignment with SIFT)

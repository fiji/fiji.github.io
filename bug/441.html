Description Davide 2012-06-23 11:48:00 CDT 
Dear Fiji-Developers,
     I am not sure whether this classifies as bug but in my view it does create un unwanted behaviour by FIJI. I recently discovered a change in the behaviour of the ROI manager of Fiji, namely that ROI events are by default not any more registered with the slice information. By investigating the cause I discovered that the ROI Manager>Option panel was changed significantly. While in previous versions it stated as option: 
associate "Show All" ROIs with the stack slices (as also discribed by the help section (http://imagej.nih.gov/ij/docs/menus/analyze.html#manager)
now it was changed to: Associate ROIs with Stack positions

The change has so far caused quite some trouble to me, first  wasting working time not realizing that the stacks were not registered  anymore and second I could not find an option that would allow to overlay of all the point marks regardless of the stack slice property.

I hope the old option can be restored or at least an alternative option to have the same viewing as in the old editions.

With best regards,
Davide

P.s. I am quite sure that the change came with the new update of Fiji, I apologise for not being able to give the specific detail. Currently I am running the latest version which does have the changed option.

Description Jan Eglinger 2013-03-08 12:11:54 CST 
I noticed that simple plugins (i.e. plugins without any plugins.config file, consisting of a single class implementing ij.plugin.PlugIn) don't work if the class file is put in a subfolder of ./plugins

To reproduce, compile the following minimal example and put the resulting class file in a subfolder of ./plugins/ : this will result in the exception pasted below. Putting the class in the main ./plugins folder (without subfolder) will work as expected.

/* code example Bare_PlugIn */
import ij.IJ;
import ij.plugin.PlugIn;

public class Bare_PlugIn implements PlugIn {
	@Override
	public void run(String arg) {
		IJ.log("Hello world!");
	}
}
/* end of example code */
